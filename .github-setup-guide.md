## 步骤 2: 配置 GitHub 个人访问令牌 (PAT)

由于 GitHub 已停止支持密码认证，你需要创建 Personal Access Token：

1. 访问 https://github.com/settings/tokens
2. 点击 **Generate new token (classic)**
3. 填写信息：
   - **Note**: `SourceTree-xunming`
   - **Expiration**: 选择 90 天或 No expiration
   - **Scopes**: 勾选以下权限：
     - ☑️ repo (完整仓库访问)
     - ☑️ workflow (如果需要 GitHub Actions)
4. 点击 **Generate token**
5. **立即复制 token**（只显示一次！）

## 步骤 3: 配置 SourceTree

### 3.1 安装 SourceTree
1. 访问 https://www.sourcetreeapp.com/
2. 下载并安装 Windows 版本
3. 首次启动时选择 "Use an existing account"

### 3.2 添加 GitHub 账户
1. 打开 SourceTree → **工具** → **选项** (Ctrl+,)
2. 选择 **验证** 标签
3. 点击 **添加** 按钮
4. 配置：
   - **托管服务商**: GitHub
   - **身份验证类型**: 个人访问令牌 (Personal Access Token)
   - **用户名**: 你的 GitHub 用户名
   - **密码**: 粘贴刚才生成的 PAT Token
5. 点击 **刷新密码** 或 **确定**

### 3.3 添加本地仓库
1. SourceTree 主界面点击 **添加/创建**
2. 选择 **添加现有本地仓库**
3. 浏览到路径：`C:\seekname`
4. 点击 **添加**

### 7.1 修改 GitHub 密码
1. 访问 https://github.com/settings/security

   - **推荐**: Authenticator app (Google Authenticator / Microsoft Authenticator)

## 常见问题

### Q: 推送时提示 "Authentication failed"
A: 确保使用 Personal Access Token 而不是密码进行认证。

### Q: 提示 "Permission denied"
A: 检查 PAT Token 是否有 `repo` 权限，或确认你是仓库的所有者/协作者。

### Q: SourceTree 无法识别 Git
A: 安装 Git for Windows: https://git-scm.com/download/win
然后在 SourceTree 选项中配置 Git 路径。
