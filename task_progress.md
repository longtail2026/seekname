# 任务进度

## 问题分析

根据日志和起名结果，发现三个核心问题：

1. **典籍出处缺少篇章名称和具体句子，也没有现代译文**
   - 新路径（naming_materials 匹配）不搜索 naming_classics，导致 result.matches 为空
   - prompt 不够强约束，AI 输出泛泛的"出自《诗经》" 

2. **名字怪异、重复、拼音错误**
   - "朗张"、"昂开"、"昂端" 等机械拼接字词
   - 重复名：张朗张、张昂开、张昂端各出现两次
   - 拼音错误：昫→Xiǎo(应为xù)、禔→Zhē(应为zhī)
   - fallback 代码未去重，把已过滤的名字又加了回来

3. **打分不准确，最终结果未按从高到低排序**
   - 过滤后不足10个时，从 generatedNames 补的全是 80 分默认值
   - 最终列表未按 score 降序排序
   - 未过滤的 fallback 名也带着一样的 score
